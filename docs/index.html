Docs

Switch your git identity without guessing what the next commit will use.

gitrole stores named git identities, applies them at global or repo-local scope, and helps you verify whether commit identity, remote configuration, and SSH auth are actually aligned before you push.

Start Here npm package

Install

Use the package directly.

npm install -g gitrole
npx gitrole --help

What it does

  • Stores named git identity profiles
  • Switches global or repo-local commit identity
  • Optionally loads an SSH key with ssh-add
  • Checks whether repo remote and SSH auth match the selected role
  • Explains the effective identity before the next commit

Quickstart

Save a role, switch to it, then verify.

Add a role before you use it. gitrole use <name> switches to a saved role; it does not create one. 

gitrole add work \
  --name "Alex Developer" \
  --email "alex@work.example" \
  --ssh ~/.ssh/id_work \
  --github-user acme-dev \
  --github-host github.com-acme-dev

gitrole use work
gitrole status
gitrole verify

Scope

Use global when you mean global. Use local when a repo must stay isolated.

gitrole use work
gitrole use work --global
gitrole use work --local
  • use <role> defaults to global scope
  • --global is explicit global scope
  • --local writes only repo-local user.name and user.email
  • --local requires being inside a git repository

Verification

See the effective identity, not just raw git config.

gitrole status
work  Alex Developer <alex@work.example>  local override  aligned
last non-merge commit  Alex Developer <alex@work.example> — docs(cli): improve terminal help text

Commands

Command surface

Command Purpose
gitrole add <name> --name "..." --email "..." Save or update a named role
gitrole use <name> [--global | --local] Apply a role globally or to the current repository
gitrole current Show which saved role matches the effective identity
gitrole current --verbose Show the active role plus repo and auth diagnostics
gitrole list List saved roles and mark the active one
gitrole status Compact human-readable alignment summary
gitrole status --short Stable one-line output for scripts and prompts
gitrole verify Confirm the next commit identity against recent non-merge history
gitrole doctor Diagnose identity, repo, remote, and SSH auth alignment
gitrole doctor --json Return the raw diagnostic result contract as JSON
gitrole remote use <name> Rewrite origin to the role’s configured SSH host alias
gitrole remove <name> Remove a saved role

Status

Fast scan

gitrole status
gitrole status --short

Use status for the compact human view. Use status --short for scripts, prompts, and CI.

Verify

Trust the next commit

gitrole verify

verify
  identity  Alex Developer <alex@work.example>
  scope     local
  role      work
  next commit Alex Developer <alex@work.example>
  last non-merge commit Alex Developer <alex@work.example> — docs(cli): improve terminal help text
  status    aligned

verify is the dedicated command for the “did the switch really take effect?” question.

SSH Host Aliases

Use GitHub host aliases for deterministic SSH identity.

Host github.com-acme-dev
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_work
  IdentitiesOnly yes

After that, gitrole remote use work can rewrite origin to use the matching host alias while preserving the owner and repo slug.

What it does not do

  • No GitHub browser or session switching
  • No gh auth, HTTPS credential, or token management
  • No interactive prompts
  • No automatic SSH config editing
