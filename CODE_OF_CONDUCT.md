# Code of Conduct

## Our Standard

This project expects contributors to be respectful, direct, and constructive.

Examples of behavior that help maintain a healthy project:

- assuming good intent while still giving precise technical feedback
- welcoming questions, bug reports, and corrections
- focusing critique on ideas, code, and behavior rather than people
- accepting responsibility for mistakes and correcting them quickly
- making room for different levels of experience

Examples of unacceptable behavior:

- harassment, intimidation, threats, or discrimination
- personal attacks, insults, or bad-faith engagement
- publishing private information without permission
- deliberately disruptive or exclusionary behavior
- sexualized language or unwelcome attention

## Scope

This code of conduct applies to project spaces, including:

- issues
- pull requests
- discussions
- code review
- any other communication channels used for project collaboration

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, issues,
and other contributions that violate this code of conduct.

Maintainers may also temporarily or permanently restrict participation for
contributors who repeatedly or seriously violate these standards.

## Reporting

If you experience or observe unacceptable behavior, report it to the project
maintainers through the repository’s private contact method or another
maintainer-approved channel.

Reports will be reviewed in good faith and handled as discreetly as practical.
